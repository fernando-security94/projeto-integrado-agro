✅ Criação de gráficos com Seaborn e Matplotlib (linhas, barras, boxplot, scatter, heatmap)

🏆 Plano de Minicurso de Férias – Análise de Dados
🔥 Foco: Python, SQL, Pandas, Visualização e Machine Learning Básico
🗓️ Estrutura Semanal com Exercícios Progressivos
🔹 Semana 1 – Automação e Manipulação de Dados com Python
✅ Objetivos:
Leitura e escrita de arquivos (CSV, Excel, JSON)

Automatizar tarefas e organizar dados

🔧 Exercícios:
Organizador de Arquivos:
- Crie um script que mova arquivos .pdf, .jpg e .xlsx para pastas específicas 
dentro de uma pasta chamada Documentos_Organizados.

Leitor de Arquivos JSON:
- Crie um arquivo dados.json com dados de clientes e leia com Python.

- Transforme o JSON em DataFrame.

Concatenação de CSVs:
- Monte uma pasta com 3 CSVs (dados de vendas de janeiro, fevereiro e março).
- Crie um script que junte todos e gere um vendas_trimestre.csv.

Conversor Excel para CSV:
- Leia uma planilha Excel e exporte cada aba para um CSV separado.

🔥 Desafio Master:
Crie um script que leia os arquivos de uma pasta, combine os dados, gere um relatório de vendas e envie automaticamente por e-mail (simulado, basta gerar o arquivo final).

🔹 Semana 2 – SQL para Análise de Dados
✅ Objetivos:
Modelagem simples de banco de dados
Consultas SQL com filtros, joins e agregações

🔧 Exercícios:
Criação do Banco:
- Monte um banco empresa.db com as tabelas funcionarios, departamentos e vendas.

Consultas Simples:
- Liste todos os funcionários do departamento de "Marketing".

Consultas com Agregações:
- Calcule o total de vendas por funcionário.

Joins:
- Crie uma consulta que mostre nome do funcionário, departamento e total de vendas.

Consultas Avançadas:
- Liste os 3 funcionários que mais venderam.

- Qual departamento teve maior faturamento?

🔥 Desafio Master:
Construa um relatório completo, extraindo dados do banco com Python + SQLite e gerando uma análise em um arquivo Excel.

🔹 Semana 3 – Análise e Tratamento de Dados (Pandas + NumPy)
✅ Objetivos:
Manipulação, limpeza e transformação de dados

🔧 Exercícios:
Limpeza de Dados:
- Crie um DataFrame com dados faltantes e outliers.

- Trate os dados preenchendo faltantes e removendo outliers.

- Agrupamentos:

- Usando dados de vendas, agrupe por produto, mês e calcule total e média.

Merge e Join com Pandas:
- Combine dois DataFrames (clientes e vendas) para criar um relatório detalhado.

Criação de Novas Colunas:
- Crie colunas calculadas como: faturamento = quantidade * preco.

Pivot Table:
- Gere uma tabela dinâmica mostrando vendas por produto e mês.

🔥 Desafio Master:
Crie uma pipeline completa:

Leia dados crus -> limpe -> transforme -> gere um relatório final (CSV e Excel).

🔹 Semana 4 – Visualização Profissional (Seaborn, Matplotlib e Tableau)
✅ Objetivos:
Criação de gráficos explicativos e dashboards

🔧 Exercícios:
Gráfico de Linha:
- Evolução do faturamento mensal.

Gráfico de Barras:
- Faturamento por produto.

Boxplot:
- Distribuição dos salários dos funcionários.

Heatmap de Correlação:
- Correlação entre idade, salário e vendas.

Dashboard no Tableau:
- Importe dados e crie um dashboard interativo com filtros por produto, mês e funcionário.

🔥 Desafio Master:
Construa um dashboard completo (1 em Python e 1 no Tableau) respondendo:

Quais são os melhores meses?

Quais os produtos mais vendidos?

Qual vendedor performa melhor?

🔹 Semana 5 – Machine Learning na Prática (Introdução)
✅ Objetivos:
Pipeline básico de machine learning supervisionado

🔧 Exercícios:
Criação de Dataset Simulado:
- Dados de clientes com idade, salário e se compraram (0/1).

Pré-processamento:
- Normalização, tratamento de dados categóricos.

Divisão de Dados:
- Split em treino e teste.

Modelagem:
- Treinar modelo de classificação (árvore de decisão, KNN ou regressão logística).

Avaliação:
- Accuracy, matriz de confusão e relatório de classificação.

🔥 Desafio Master:
Escolha um dataset do Kaggle (ex.: Titanic ou Iris) e execute todo o pipeline de ML:

Exploração -> Limpeza -> Modelagem -> Avaliação -> Relatório final.

Começar a aprender Web Scraping (BeautifulSoup)

Dashboards Interativos com Streamlit

🎯 Dicas avançadas:
Nunca use conn.commit() em operações de leitura (SELECT), 
pois isso serve para gravar alterações no banco (INSERT, UPDATE, DELETE).
Sempre utilize AS para nomear suas colunas de agregação. Isso melhora muito a legibilidade e clareza do código.
